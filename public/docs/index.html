Introduction

Base URL: https://refer-friend.tw-dev.com

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer {YOUR_AUTH_KEY}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your API page Generate API key.

Endpoints

Upsert Referred Friend's Status

requires authentication

Uses the match_* fields to update a Referred friend, and sets the status. If no match is found a new Referred Friend will be created with the given status.

Example request:
curl --request POST \
    "https://refer-friend.tw-dev.com/api/referred/upsert" \
    --header "Authorization: Bearer {YOUR_AUTH_KEY}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"referees\": [
        {
            \"match_email\": \"gerlach.johathan@example.com\",
            \"match_phone_number\": \"+61 5054 43251\",
            \"match_first_name\": \"cyviiatfvmmscxhr\",
            \"match_last_name\": \"eotjtmrfrfb\",
            \"new_status\": \"loan_eligible\"
        }
    ]
}"

const url = new URL(
    "https://refer-friend.tw-dev.com/api/referred/upsert"
);

const headers = {
    "Authorization": "Bearer {YOUR_AUTH_KEY}",
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "referees": [
        {
            "match_email": "gerlach.johathan@example.com",
            "match_phone_number": "+61 5054 43251",
            "match_first_name": "cyviiatfvmmscxhr",
            "match_last_name": "eotjtmrfrfb",
            "new_status": "loan_eligible"
        }
    ]
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (201):


{
    "message": "Upsert Successful"
}

Request      

POST api/referred/upsert

Headers

Authorization      

Example: Bearer {YOUR_AUTH_KEY}

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

referees   object[]   

the list of Referred friends to upsert
match_email   string   

Must be a valid email address. Example: gerlach.johathan@example.com

match_phone_number   string   

Must be at least 10 characters. Must not be greater than 50 characters. Example: +61 5054 43251

match_first_name   string   

Must be at least 2 characters. Must not be greater than 50 characters. Example: cyviiatfvmmscxhr

match_last_name   string   

Must be at least 2 characters. Must not be greater than 50 characters. Example: eotjtmrfrfb

new_status   string   

Must be one of form_completed, not_interested, meeting_booked, ineligible, pension_boost_eligible, loan_eligible, loan_approved, or pension_boost_approved. Example: loan_eligible

Create New Referrers

requires authentication

creates new referrer from an email address. this referrer will be email a form to refer friends with

Example request:
curl --request POST \
    "https://refer-friend.tw-dev.com/api/referrer/create" \
    --header "Authorization: Bearer {YOUR_AUTH_KEY}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"referrers\": [
        \"quidem\"
    ]
}"

const url = new URL(
